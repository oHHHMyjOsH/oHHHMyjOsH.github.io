怪异模式和标准模式

# 简言

怪异模式/兼容模式 (quirks mode) 和 标准模式 (strict mode)，简单来说就是两种不同的排版 / 渲染方式，前者按 W3C 标准 解析执行代码，后者使用 浏览器自己的方式 解析执行代码。浏览器依据文件开头的 DOCTYPE 来判断是何种模式并处理。

其他：

  • doctype 是一种标准通用标记语言的文档类型声明，目的是告诉标准通用标记语言解析器要使用什么样的文档类型定义（DTD）来解析文档
  • 如 H5 文档就是使用的 <!DCOTYPE HTML>
  • DOCTYPE 一定要放在第一行 (否则可能导致早期浏览器触发怪异模式)
  • 对于 XHTML 而言则没有这种困扰，它在 Content-Type HTTP 标头使用 application/xhtml+xml MIME 类型，不需要使用 DOCTYPE ，因为这种文件会永远使用标准模式。

# 历史背景

根据 MDN，在很久以前的网络上，页面通常有两种版本：

  1. 为网景（Netscape）的 Navigator 准备的版本，
  2. 为微软（Microsoft）的 IE 准备的版本。

当 W3C 创立网络标准后，为了不破坏当时既有的网站，浏览器不能直接弃用这些标准。因此，浏览器采用了两种模式，用以把能符合新规范的网站和老旧网站区分开。

除了这两种 "排版引擎" 以外，还有接近标准模式 (Almost standards mode)。在接近标准模式下，只有少数的怪异行为被实现。

# 两者区别

  1. 标准模式元素的 width、height 是 content 的宽、高 (内容宽高)，而怪异模式中元素的 width、height 均包含了 border 和 padding (元素实际宽高，又称作 怪异盒 / IE盒 );

  2. 标准模式中给行内元素如 span 设置宽高无效，而怪异模式中是会生效的；

  3. 百分比问题：

    CSS 中对于元素的百分比高度规定：百分比为元素包含块的高度，不可为负值；如果包含块的高度没有显示给出，该值等同于 auto，所以百分比的高度必须是在元素有高度声明的情况下使用。

    标准模式下，高度取决于内容变化，怪异模式下，百分比高度被准确应用

  4. 水平居中方式不同：在标准模式下 margin: 0 auto 可以使元素水平居中，但是在怪异模式下，无效，可以用 text-align : center; 来实现居中；

  5. 怪异模式下设置图片的 padding 会失效；

  6. 怪异模式下 table 中的字体属性不能继承上层的设置；

  7. 溢出处理不同：标准模式下，overflow 默认取 visible，在怪异模式下，溢出会被当做扩展 box 来对待，溢出不会裁减，元素框自动调整大小，能够包含溢出内容.
